## 一、产品概述

TokenCounter 是一款 Windows 系统托盘工具，用户通过双击 Ctrl 键即可快速统计所选文本的 token 数量和字符数，结果以浮动气泡（tooltip）形式显示在光标附近。支持 OpenAI tiktoken 的 o200k_base（GPT-4o）和 cl100k_base（GPT-4/3.5）两种分词模型，面向需要频繁估算 token 用量的 AI 开发者与内容创作者。

## 二、现有功能与用户可调设置汇总（v0.1.0）

### 2.1 功能清单

| # | 功能 | 说明 |
|---|------|------|
| F1 | 双击 Ctrl 触发 | 检测双击左 Ctrl（≤500ms 间隔），模拟 Ctrl+C 获取选中文本并计算 token |
| F2 | Token 计数浮动气泡 | 深色主题气泡，显示 token 数（青色）、字符数（蓝色）、模型名（橙色），支持淡出动画 |
| F3 | 鼠标悬停暂停淡出 | 气泡淡出过程中鼠标移入将暂停消失 |
| F4 | 系统托盘图标 | 右键菜单 + 双击打开控制面板 |
| F5 | 控制面板 (Tkinter) | 独立设置窗口，支持浅色/深色主题跟随系统 |
| F6 | 从剪贴板计算 | 无需选中文本，直接对剪贴板内容计算 token |
| F7 | 启动动画 | 应用启动时气泡从屏幕中央弹出，显示 "TokenCounter — Running in system tray" |
| F8 | 开机启动 | 通过 Windows 注册表实现自启动 |
| F9 | 进程黑名单 | 在指定进程的前台窗口中不触发 |
| F10 | 去重窗口 | 2 秒内相同文本不重复弹出 |
| F11 | 日志记录 | 日志写入 `%APPDATA%\TokenCounter\tokencounter.log` |

### 2.2 用户可调设置

| 设置项 | 默认值 | GUI 入口 | 配置文件字段 | 说明 |
|--------|--------|----------|-------------|------|
| **启用/禁用** | 启用 | 控制面板开关 / 托盘菜单 | `enabled` | 全局开关 |
| **分词模型** | o200k_base | 控制面板下拉 / 托盘子菜单 | `tokenizer` | o200k_base (GPT-4o) 或 cl100k_base (GPT-4) |
| **气泡显示时长** | 2 秒 | 控制面板滑块(1-5s) / 托盘子菜单(1/2/3/5秒) | `tooltip_display_s` | 气泡在淡出前的驻留时间 |
| **开机启动** | 关 | 控制面板开关 / 托盘菜单 | — (注册表) | Windows 登录时自动运行 |
| **进程黑名单** | 空 | 控制面板文本区 | `blacklist` | 排除特定 .exe 进程 |
| **触发热键** | 左 Ctrl (0xA2) | ❌ 无 GUI | `hotkey_vk` | 仅可通过编辑 config.json 修改 |
| **触发模式** | hotkey | ❌ 无 GUI / 锁定 | `trigger_mode` | 当前锁定为 "hotkey"，不可修改 |

### 2.3 配置存储

- 路径: `%APPDATA%\TokenCounter\config.json`
- 原子写入（tmp + rename），加载容错
- 线程安全（ConfigManager 内部带锁）

## 三、现有问题与改进空间

| # | 类别 | 问题 | 影响 |
|---|------|------|------|
| P1 | 易用性 | 触发热键（hotkey_vk）无法在 GUI 中修改，新用户不知道可以改 | 需编辑 JSON 文件，门槛高 |
| P2 | 易用性 | 托盘右键菜单部分为中文（"1秒/2秒"）、部分为英文，语言混杂 | 体验不一致 |
| P3 | 功能 | 只支持 2 种 tiktoken 编码，无法自定义或扩展 | 无法满足使用 Claude / Gemini / 国产模型的用户 |
| P4 | 反馈 | 选中失败（空文本）时没有任何反馈 | 用户不知道是否触发成功 |
| P5 | 可访问性 | 气泡不支持键盘操作或屏幕阅读器 | 无障碍缺失 |
| P6 | 可靠性 | 模拟 Ctrl+C 会覆盖用户剪贴板内容 | 用户可能丢失剪贴板数据 |

## 四、v0.2.0 新需求提案

### 需求 R1：自定义触发热键（优先级：高）

**背景**: 当前触发键固定为双击左 Ctrl，部分用户习惯不同，或与其他软件冲突。  
**需求描述**:  
- 在控制面板中新增「触发热键」设置区域
- 支持录制热键：用户点击「录制」按钮后按下想要的按键组合，捕获并保存
- 至少支持单键双击模式（如 Ctrl / Alt / Shift）
- 扩展支持组合键模式（如 Ctrl+Shift+T 单次按下触发）
- 修改后实时生效，无需重启

**验收标准**:  
1. 用户可通过控制面板 GUI 修改触发热键
2. 修改后立即生效
3. config.json 中正确持久化新热键值
4. 无效/冲突按键给出提示

---

### 需求 R2：剪贴板保护（优先级：高）

**背景**: 当前触发时通过模拟 Ctrl+C 获取选中文本，会覆盖用户原有剪贴板内容，在某些工作流中造成数据丢失。  
**需求描述**:  
- 触发前备份当前剪贴板内容
- 获取选中文本后恢复原有剪贴板内容
- 支持文本（CF_UNICODETEXT）及富文本（CF_HTML）格式的备份/恢复
- 若恢复失败，记录日志但不中断流程

**验收标准**:  
1. 触发 token 统计后，用户 Ctrl+V 粘贴的内容与触发前一致
2. 对含图片/文件的剪贴板内容不产生破坏（graceful skip）

---

### 需求 R3：多模型对比显示（优先级：中）

**背景**: 不同模型的 token 数差异较大，用户经常需要知道同一段文本在多个模型下的 token 数。  
**需求描述**:  
- 在控制面板中支持勾选多个模型（最多 3 个同时生效）
- 气泡同时显示多行结果，每行一个模型的 token 数
- 气泡高度自适应行数
- 配置字段从 `tokenizer: str` 扩展为 `tokenizers: list[str]`（向后兼容单值字符串）

**验收标准**:  
1. 勾选 2 个模型时，气泡显示 2 行 token 结果
2. 气泡尺寸自动适配，不截断
3. 旧版 config.json（`tokenizer` 单值）可被正确迁移

---

### 需求 R4：国际化 (i18n)（优先级：中）

**背景**: 当前 UI 文字中英混杂，托盘菜单有中文（"1秒"），控制面板为英文。  
**需求描述**:  
- 建立国际化框架，将所有 UI 字符串抽取到语言文件
- 首批支持简体中文 (zh-CN) 和英文 (en-US)
- 语言跟随系统区域设置，或在控制面板中手动切换
- 新增配置项 `language: str = "auto"`

**验收标准**:  
1. 所有 UI 元素（托盘菜单、控制面板、气泡、通知）语言一致
2. 切换语言后无需重启应用
3. 语言文件结构清晰，便于后续添加新语种

---

### 需求 R5：历史记录面板（优先级：低）

**背景**: 用户有时需要回顾之前统计过的 token 数据，当前无记录功能。  
**需求描述**:  
- 记录最近 50 条统计结果（时间戳、文本摘要前 50 字符、token 数、字符数、模型）
- 在控制面板新增「历史记录」标签页
- 支持点击单条记录查看完整文本
- 数据仅存内存，关闭应用后清空（不涉及隐私持久化）

**验收标准**:  
1. 每次触发统计后，历史列表自动更新
2. 列表显示时间、文本摘要、token 数
3. 超过 50 条时自动淘汰最旧记录

---

### 需求 R6：气泡点击复制（优先级：低）

**背景**: 用户看到 token 数后经常需要粘贴到别处（如 API 成本计算表），目前需手动记忆数字。  
**需求描述**:  
- 单击气泡时将 token 数复制到剪贴板（纯文本，如 `1,234`）
- 气泡短暂闪烁或变色反馈复制成功
- 复制格式可配置（纯数字 `1234` / 带千分位 `1,234`）

**验收标准**:  
1. 单击气泡后，Ctrl+V 可粘贴 token 数文本
2. 有明显的视觉反馈（颜色变化或闪烁）

---

### 需求 R7：选中失败反馈（优先级：低）

**背景**: 双击 Ctrl 后如果未选中任何文本，不会有任何反馈，用户不确定功能是否正常。  
**需求描述**:  
- 当获取文本为空时，显示简短气泡提示（如 "未检测到选中文本"）
- 该提示 1 秒自动消失，视觉样式区别于正常结果（如灰色调）
- 可在设置中关闭此提示

**验收标准**:  
1. 未选中文本时显示提示气泡
2. 提示样式与正常结果有视觉区分
3. 不干扰连续快速触发的体验

## 六、技术约束与注意事项

1. **64 位 ctypes 兼容性**: 所有 Win32 API 指针类型必须声明 `restype = c_void_p`，否则在 64 位系统上截断导致崩溃（已有前车之鉴）
2. **PyInstaller 打包**: `hMod` 参数必须传 `0`，`GetModuleHandleW(None)` 在打包环境下返回值不可靠
3. **线程安全**: Config 修改须通过 ConfigManager（内置锁），UI 更新须通过各线程的命令队列
4. **剪贴板竞争**: 剪贴板操作有重试机制（CLIPBOARD_COPY_TIMEOUT_S = 0.8s），需注意与剪贴板保护功能的交互
5. **Tkinter 线程约束**: ControlPanel 运行在独立线程，所有 Tk 操作必须通过 `after()` 调度到主循环

```json
{
  "tokenizer": "o200k_base",
  "trigger_mode": "hotkey",
  "hotkey_vk": 162,
  "enabled": true,

  "blacklist": []
}
```

```
src/tokencounter/
├── __init__.py
├── __main__.py          # 入口点
├── app.py               # 应用编排器

├── config.py            # 配置管理
├── constants.py         # 常量定义
├── control_panel.py     # Tkinter 控制面板

```
